# FIND — Terminal Cheat Sheet

Every command you need, by area. Run from the repo root unless noted.
Python commands assume the repo venv (`.venv/bin/python` or an activated venv).

## First-run bootstrap

| Command | What it does |
|---|---|
| `python scripts/bootstrap_ledger.py` | Creates the ledger / audit / authorization SQLite DBs. Run once before first start. |
| `python scripts/seed_identity.py` | Seeds the identity vault with starter accounts. |
| `python scripts/seed_prospects.py` | Seeds the sales pipeline with prospects. |
| `cp .env.example .env` | Config template — mock connectors work with no secrets filled in. |

## Run it

| Command | What it does |
|---|---|
| `python scripts/run_dev.py` | Dev mode: uvicorn backend on **:8000** + Vite frontend on **:5173** together. |
| `python -m uvicorn find.main:app --reload --port 8000` | Backend only (serves built SPA from `frontend/dist/` if present). |
| `npm run dev --prefix frontend` | Frontend only (proxies `/api` + `/ws` to :8000). |
| `npm run build --prefix frontend` | Typecheck (`tsc`) + production build → `frontend/dist/`. |
| `docker compose up` | Containerized run (see `DEPLOY.md`). |
| `curl localhost:8000/health` | Liveness check. |

## Tests & CI

| Command | What it does |
|---|---|
| `pytest` | Full backend suite (`tests/`, config in `pyproject.toml`). |
| `pytest tests/test_votes_api.py -q` | Just the ballot/branch-lock tests. |
| `npx tsc --noEmit` (in `frontend/`) | Frontend typecheck without building. |

## The clock (scheduler)

| Command | What it does |
|---|---|
| `curl localhost:8000/api/scheduler` | Running state, timezone, job count. |
| `curl localhost:8000/api/scheduler/jobs` | Every cron + one-shot job with next-fire times (the Clock page's data). |

The job definitions live in `find/scheduler/jobs.py`; the store is
`data/scheduler.db`. Jobs also fire agents you can trigger by hand:

| Command | What it does |
|---|---|
| `curl -X POST localhost:8000/api/agents/FinanceAgent/run` | Manual agent trigger (any name from `GET /api/agents`). |
| `curl localhost:8000/api/agents` | All agents with last-run status and 24h cost. |

## Votes (branching season)

| Command | What it does |
|---|---|
| `curl localhost:8000/api/votes/1/6/results` | Participation (open) or full tally + winner (closed). |
| `curl -X POST localhost:8000/api/votes/1/6 -d '{"patron_id":"p1","choice_id":"A","branch_id":"branch_a"}' -H 'Content-Type: application/json'` | Cast a ballot (idempotent per patron). |
| `curl -X POST localhost:8000/api/votes/1/6/close` | Close + tally + write the Heterodyne `branch_lock.json`. **Canon-setting.** |

## Fleet & lore

| Command | What it does |
|---|---|
| `curl localhost:8000/api/fleet/models` | SLM fleet: backends, pricing, capabilities, routing table, fallbacks. |
| `curl localhost:8000/api/lore` | The Heterodyne lore export served to Auralbouros (404 until the export script has run; path: `HETERODYNE_LORE_EXPORT_PATH`). |

## Where things live

- Config: `find/config.py` (+ `.env`); fleet manifest: `config/llm_routing.yaml`
- DBs: `data/*.db` (ledger, audit, auth, scheduler, votes)
- Frontend pages: `frontend/src/pages/` — Queue, Todos, Status, Finance,
  Customers, Pipeline, Subscriptions, Documents, Calendar, Agents, Clock,
  HETERODYNE, Votes, Board, Audit
