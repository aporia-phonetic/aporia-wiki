# Auralbouros — Terminal Cheat Sheet

Expo / React Native app. Run from the repo root; Node ≥ 18.

## Dev

| Command | What it does |
|---|---|
| `npm install` | Install deps (respect `package-lock.json`). |
| `npm start` | Expo dev server (QR for Expo Go; press `a` / `i` for emulators). |
| `npm run android` | Expo dev server + open Android emulator. |
| `npm run ios` | Expo dev server + open iOS simulator (macOS only). |
| `npm run typecheck` | `tsc --noEmit` — the same gate CI runs. |

## Builds

| Command | What it does |
|---|---|
| `npm run build:android` | `eas build --platform android` (cloud build; config in `eas.json`, needs `eas login`). |
| `npx eas build --platform ios` | iOS cloud build (needs Apple Developer account). |
| `npx eas submit --platform android` | Store submission once builds exist. |

## CI gates (run these before pushing)

| Command | What it does |
|---|---|
| `npm run typecheck` | Type errors fail CI (`.github/workflows/ci.yml`). |
| `grep -rn '<<<<<<<' src/ docs/ *.md` | CI also greps for committed merge-conflict markers — so should you. |

## Backend wiring

- **Lore feed** — `src/services/lore.ts` fetches FIND's `GET /api/lore`
  (falls back to `stubLore.ts` offline).
- **Votes** — `src/services/votes.ts` POSTs ballots to FIND's
  `/api/votes/{season}/{episode}`; the API base URL is editable in the app's
  Settings screen.
- **CYOA cartridges** — `src/services/cyoaCartridge.ts` loads Suit-Yourself
  content bundles.
