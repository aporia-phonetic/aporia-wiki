# Godot Pipeline — Terminal Cheat Sheet

Offline animation pipeline: HETERODYNE episode JSON → clip specs → Godot 4
Movie Maker → MP4. Run from the repo root. **Project status: parked** — see
`STATUS.md` before investing effort here.

## Compositor (Python, stdlib-only)

| Command | What it does |
|---|---|
| `python3 compositor/episode_to_scene.py <episode.json>` | Parse an episode, emit per-clip specs + `out/S{S}E{EE}_manifest.json`. |
| `python3 compositor/episode_to_scene.py <ep.json> --audio-root ../../HETERODYNE/audio --out out --world-id <id>` | Full form: audio location, output dir, world id (defaults shown; world id falls back to `verdant_deep`). |

## Renderer (Godot 4, headless)

| Command | What it does |
|---|---|
| `godot --headless --path project/ --write-movie out/clip.mp4 -- render.gd compositor/out/S1E01_manifest.json` | Render every clip in the manifest to its own MP4 via Movie Maker mode. |

## Tests

| Command | What it does |
|---|---|
| `python3 tests/test_episode_to_scene.py` | Compositor unit tests (also runnable as `pytest`). |

## Prerequisites & gotchas

- **Godot 4.x** with Movie Maker mode (any 4.x; version unpinned — pin one
  before real production use).
- **Assets not bundled**: `assets/sprites/` and `assets/scene_templates/`
  are `.gitkeep` placeholders; LoRA weights go in `assets/loras/`
  (gitignored). A render without assets has nothing to draw.
- **`out/` is disposable** — gitignored, regenerable, never source of truth.
- 9:16 TikTok/Reels clips need a separate render configuration from the
  standard-format clips.
