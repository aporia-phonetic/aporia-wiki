# SOPHROSYNE — Terminal Cheat Sheet

Personal honesty engine (formerly LODESTAR). TypeScript, Node ≥ 20. Run from
the repo root. Standalone — not part of the APORIA launch trio.

## Talk to it

| Command | What it does |
|---|---|
| `npm run harness` | Text loop through the core — your first way to talk to the engine. |
| `npm run observe` | Autonomous workspace calibration: memory + docs + git. |
| `npm run observe -- --full` | The above + every GitHub repo (`gh` must be authed) + the MCP inbox digests. |

## Backup / restore

| Command | What it does |
|---|---|
| `SOPHROSYNE_BACKUP_PASS=<≥16 chars> npm run backup -- --to <path>.ldbak` | Passphrase-wrapped encrypted backup (`SOPHROSYNE-BACKUP-1` format). Passphrase via env only — never on the CLI. |
| `SOPHROSYNE_BACKUP_PASS=... npm run restore -- --from <path>.ldbak [--force]` | Restore (cross-platform: scrypt/pbkdf2 KDF union in the header). |

## Tests & checks

| Command | What it does |
|---|---|
| `npm test` | Vitest suite (16 files). |
| `npm run typecheck` | Core typecheck. |
| `npm run typecheck:rn` | React-Native shell typecheck. |
| `npm run eval` | Observer eval harness (`evals/observer/`). |

## Mobile shell (Expo)

| Command | What it does |
|---|---|
| `npm start` / `npm run android` / `npm run ios` | Expo dev server / emulators. Screens: index, settings, audit. |

## Environment

| Var | Used by | Meaning |
|---|---|---|
| `SOPHROSYNE_WORKSPACE` | observe / backup / restore | Workspace root (default is the author's `C:\APORIA`; set it on any other machine). |
| `SOPHROSYNE_MEMORY` | observe | Memory dir for durable-fact extraction. |
| `SOPHROSYNE_BACKUP_PASS` | backup / restore | Wrap-key passphrase, ≥ 16 chars, env-only. |

Data lives at `<workspace>/SOPHROSYNE/data/` (`ledger.ldb` encrypted ledger,
`ledger.key` dev key). Gmail/Drive/Calendar arrive as JSONL digests dropped
into `<workspace>/SOPHROSYNE/data/inbox/<source>/` (see `MCP_SETUP.md`) —
the engine never gets Google write access.
