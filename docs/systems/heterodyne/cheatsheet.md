# HETERODYNE — Terminal Cheat Sheet

Every part of the engine, by subsystem. Run from the repo root with the venv
active (`venv/bin/python` or `source venv/bin/activate`). `main.py` is a
single dispatcher — flags compose (e.g. `--full-run --dry-run`). `ep` targets
accept an episode JSON path or `latest`.

## Console (GUI)

| Command | What it does |
|---|---|
| `python -m gui.app` | Launch the Production Console (PyQt6). |
| `python scripts/launch_console.py` | Same, via the launcher (Windows: `launch_console.bat`). |

> `scripts/production_console.py` is the **deprecated** legacy console —
> don't edit it; the live one is the `gui/` package.

## Episode pipeline

| Command | What it does |
|---|---|
| `python main.py --full-run` | All five stages: generate → voice → music → mix. |
| `python main.py --dry-run` | Validate wiring, zero API calls. |
| `python main.py --smoke` | Smoke suite: dry-run + unit tests + fixture replay. <30 s, $0. |
| `python main.py --full-run --pipeline writers-room` | Decomposed script pipeline (beat planner → intentions → scene writer → annotators → extractor). |
| `python main.py --quality-tier premiere` | Adds scene critic + revision pass + blind voice test. |
| `python main.py --review-gates` | Season gate-chain critic → `plot_gates.proposed.json` (read-only w.r.t. `plot_gates.json`). |
| `python main.py --review-gates --refine-gates` | Also apply proposed rewrites for showrunner approval. |
| `python main.py --parallax-qa` | Dual-validation QA after each segment (~3 calls/segment). |
| `python main.py --repair-commit` | Recover paid output from checkpoints after a Literal-coercion commit failure. |

Env toggles read by the generate path: `VOICE_SENTINEL_ENABLED`,
`EMOTION_JUDGE_ENABLED`, `LEITMOTIFS_ENABLED` (0/1, default off),
`SCRIPT_PIPELINE`, `QUALITY_TIER`, `PARALLAX_QA`, `NARRATOR_PRESENCE`.

## Audio stages (individually)

| Command | What it does |
|---|---|
| `python main.py --render-audio latest` | Phase 2: per-block WAVs. |
| `python main.py --music latest` | Phase 4B: Maestro music cue list. |

## Scheduler (the daily clock)

| Command | What it does |
|---|---|
| `python main.py --scheduler` | APScheduler daemon: at `DAILY_TRIGGER_TIME` (HH:MM, `.env`), run generate → voice → music → mix for the next episode. Blocks until interrupted. |

Also available in the console: **Story & Seasons → ⏰ Automation** (in-app
trigger + buffer gating + this daemon's start/stop).

## Worlds & lore

| Command | What it does |
|---|---|
| `python aeon.py world create <world_id> --genre <genre>` | Full world build: history → sourcebook → QA gate → ledger init. **Preferred for new worlds.** |
| `python main.py --build-world` | Same underlying pipeline for the active world (`WORLD_ID`). |
| `python aeon.py world list` / `show <id>` | Build status / print a sourcebook. |
| `python aeon.py world chronicle <id>` | Just the Sediment Engine deep-history step (expensive). |
| `python aeon.py world sourcebook <id> [--season N]` | Compile/refresh a sourcebook. |
| `python aeon.py world ingest <id> [--season N] [--baseline\|--clear SECTION\|--clear-all]` | Capture human edits to a compiled sourcebook.md as canon overrides — they survive the next recompile and get injected into episode context as highest-authority canon. `--baseline` bootstraps a pre-ingest file; `--clear`/`--clear-all` revert section(s) to machine-generated text. |
| `python main.py --chronicle` / `--sourcebook` | Flag-form equivalents for the active world. |
| `python main.py --export-history [out.md]` | Read-only Markdown export of the world history (no API key). |
| `python main.py --manifests <CHARACTER>` | A.C.E.S. outcome manifest for one character ($5–15). |
| `python scripts/export_lore.py` | Write the lore export consumed by FIND's `/api/lore` → Auralbouros. |
| `python scripts/bootstrap_world.py` / `bootstrap_season.py` | Scripted world/season bootstrap. |
| `python scripts/md_to_world_seed.py <seed.md>` | Markdown → WorldSeed JSON (template: `documents/USER MANUAL/_world_seed_template.md`). |
